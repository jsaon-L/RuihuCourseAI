# 幼儿课程课件系统低保真视觉稿

版本：1.0  
用途：第一版 MVP 界面落地参考  
范围：课程播放端、课程编辑端、幼儿课程通用环节  

## 1. 视觉稿定位

本文档是低保真视觉稿，不是最终美术稿。目标是先明确页面结构、信息层级、控件位置、课堂展示方式和编辑工作区布局，方便 Unity UGUI 开发。

第一版视觉原则：

- 播放端默认全屏。
- 优先适配 16:9 屏幕。
- 设计基准分辨率为 1280x720。
- 同时兼容 1920x1080。
- 播放端按钮大、文字大、操作少。
- 编辑端信息密度更高，优先提升制作效率。
- 第一版互动为展示型互动，不做真实答题、拖拽判定或成绩记录。

## 2. 通用布局规范

### 2.1 播放端基准

| 项目 | 建议 |
|---|---|
| 画布比例 | 16:9 |
| 基准分辨率 | 1280x720 |
| 默认模式 | 全屏 |
| 主展示区 | 尽量占据屏幕 80% 以上 |
| 老师控制区 | 默认底部浮层，可隐藏 |
| 主要按钮高度 | 64px 至 88px |
| 次要按钮高度 | 48px 至 56px |
| 儿童可见标题 | 40px 至 56px |
| 儿童可见正文 | 28px 至 36px |
| 老师辅助文字 | 20px 至 26px |

### 2.2 编辑端基准

| 项目 | 建议 |
|---|---|
| 画布比例 | 16:9 |
| 基准分辨率 | 1280x720 |
| 默认模式 | 窗口化或最大化，播放预览可全屏 |
| 左侧区域 | 导航、课程列表、环节列表 |
| 中间区域 | 表单编辑或资源管理 |
| 右侧区域 | 预览、属性、校验结果 |
| 表单标签 | 左对齐 |
| 错误提示 | 靠近出错字段显示 |

## 6. 组件清单

## 7. 第一版视觉落地边界

第一版需要做到：

- 页面结构稳定。
- 控件位置清晰。
- 播放端适合触屏。
- 编辑端支持完整制作流程。
- 所有环节均有可用展示版式。
- 互动内容明确是展示型。
- 异常弹窗不阻塞老师继续上课。

第一版暂不要求：

- 品牌级视觉精修。
- 专属插画系统。
- 复杂动效编排。
- 皮肤主题切换。
- 自定义页面布局编辑器。
- 多分辨率逐像素视觉还原。
