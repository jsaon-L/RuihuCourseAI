# 幼儿课程课件系统详细需求文档

版本：1.0  
来源：基于《幼儿课程课件系统设计大纲.md》细化  
适用阶段：第一版 MVP 开发与验收  

## 1. 文档目标

本文档将原设计大纲细化为可执行、可验收、可拆分开发的需求说明，用于指导 Unity 5.6 离线幼儿课程课件系统第一版开发。

系统包含两个独立软件：

- 课程播放端：课堂现场使用，用于读取本地课程包并播放课程。
- 课程编辑端：课程制作老师使用，用于创建、编辑、预览、校验并导出离线课程包。

两个软件使用同一个 Unity 工程开发，共享课程包结构、JSON 配置格式、资源读取规则和部分播放预览逻辑。

## 2. 产品范围

### 2.1 第一版目标

第一版目标是完成一个稳定可用的离线课程课件系统，优先保证：

- 老师能在 Windows 设备上离线播放课程。
- 课程制作老师能离线制作课程包。
- 播放端能稳定播放本地 MP4 视频、展示图片和文字。
- 制作端能校验资源缺失和视频兼容性。
- 课程包结构简单，便于复制、交付和排查问题。
- 幼儿课程环节保持少量、高通用性、可复用。

### 2.2 第一版不做范围

第一版不包含：

- 云端课程库。
- 在线课程下载。
- 在线课程更新。
- 用户账号和权限系统。
- 学生学习记录。
- 多设备同步。
- 家长端反馈。
- 在线素材市场。
- 多人协作编辑。
- 单独音频资源。
- 复杂互动游戏逻辑。
- 自动判题和成绩统计。
- 真实拖拽、连线、多人触控答题。
- TextMesh Pro。
- XML 配置。

## 3. 用户角色

| 角色 | 使用软件 | 主要目标 |
|---|---|---|
| 幼儿课堂老师 | 课程播放端 | 快速选课、稳定播放、课堂中控制课程节奏 |
| 课程制作老师 | 课程编辑端 | 创建课程、导入资源、配置环节、预览并导出课程包 |
| 教研/内容负责人 | 课程编辑端 | 检查课程完整性、确认内容可交付 |
| 幼儿学生 | 非直接操作用户 | 观看课程内容，在线下由老师引导互动 |

## 4. 运行环境与技术约束

### 4.1 运行环境

- Windows 触屏一体机。
- Windows 普通电脑。
- 老旧 Windows 笔记本。
- 最低支持 Windows 7，目标为 Windows 7 及以上桌面系统。
- 本地离线课程资源包。

### 4.2 技术约束

- 开发引擎：Unity 5.6。
- UI 系统：Unity UGUI。
- 文本组件：Unity UGUI 默认 Text。
- 不使用 TextMesh Pro。
- 配置格式：JSON。
- JSON 解析库：LitJSON。
- 视频播放：AVPro Video 1。
- 动画插件：DOTween。
- 视频转码工具：FFmpeg。
- FFmpeg 相关 DLL 和可执行依赖统一放入 `StreamingAssets/ffmpeg` 目录。
- 软件运行过程中不得依赖互联网。

### 4.4 已确认开发决策

| 编号 | 决策项 | 结论 |
|---|---|---|
| DEC-001 | Windows 支持范围 | 支持 Windows 7 及以上桌面系统。 |
| DEC-002 | FFmpeg 放置方式 | 将 FFmpeg DLL 和相关依赖放入 `StreamingAssets/ffmpeg` 目录，编辑端从该目录查找并调用。 |
| DEC-003 | 播放端启动显示模式 | 播放端默认全屏运行。 |
| DEC-004 | 第一版互动能力 | 只做展示型互动，不做真实作答、自动判题、拖拽判定或成绩记录。 |
| DEC-005 | 课程包协议状态 | 第一版开发期间协议不冻结，允许边开发边调整；调整时需同步更新播放端、编辑端和本文档。 |
| DEC-006 | 视觉稿状态 | 当前缺少正式视觉稿，先使用《幼儿课程课件系统低保真视觉稿.md》作为第一版界面落地依据。 |

### 4.3 工程结构

- 使用一个 Unity 工程。
- 同一套公共代码和公共资源。
- 两套启动入口。
- 两套场景流程。
- 两个独立打包目标。
- 播放端和编辑端共享课程配置格式。
- 播放端和编辑端共享资源读取规则。

## 5. 总体业务流程

### 5.1 课程制作到播放闭环

```text
课程制作老师打开编辑端
  ↓
新建或打开课程草稿
  ↓
填写课程基础信息
  ↓
导入封面、图片、视频资源
  ↓
配置课程环节
  ↓
预览单个环节或完整课程
  ↓
校验资源和视频格式
  ↓
导出离线课程包
  ↓
将课程包复制到播放端 Courses 目录
  ↓
课堂老师打开播放端
  ↓
选择课程并播放
```

### 5.2 数据闭环要求

- 编辑端导出的课程包必须能被播放端直接读取。
- 课程包移动到不同本地目录后仍应可播放。
- JSON 中不得保存制作电脑上的绝对路径。
- 播放端遇到损坏课程包时，应跳过该课程，不影响其他课程。

## 6. 课程播放端详细需求

### 6.1 播放端定位

课程播放端是面向幼儿线下课堂的 Windows 离线播放程序，用于读取本地课程包，并按照课程环节顺序播放视频、图片、文字提示和展示型互动内容。

第一版播放端优先目标是稳定、清晰、易操作，优先保证老旧 Windows 电脑和触屏一体机上的视频播放稳定性。

### 6.2 播放端页面

| 页面 | 说明 |
|---|---|
| 启动加载页 | 启动程序、初始化设置、扫描课程目录 |
| 课程列表页 | 展示本地有效课程包 |
| 课程播放页 | 播放视频、图片、文字和互动展示内容 |
| 环节选择面板 | 展示课程所有环节并支持跳转 |
| 异常提示弹窗 | 显示资源缺失、视频失败等错误 |
| 退出确认弹窗 | 对返回课程列表、退出程序等高风险操作二次确认 |

### 6.3 播放端主流程

```text
启动播放端
  ↓
扫描程序同级 Courses 目录
  ↓
读取各课程包 manifest.json
  ↓
展示课程列表
  ↓
老师选择课程
  ↓
读取 course.json
  ↓
按 sections.order 播放课程
  ↓
课程结束
  ↓
显示结果反馈，允许重播或返回课程列表
```

### 6.9 播放端本地记录

播放端只记录必要的本地使用数据，不记录学生个人数据。

可记录内容：

- 最近播放课程编号。
- 最近播放课程名称。
- 最近播放时间。
- 最近播放环节序号。
- 播放端基础设置，例如全屏状态。

本地记录失败不得影响课程播放。

## 7. 课程编辑端详细需求

### 7.1 编辑端定位

课程编辑端用于在离线 Windows 环境中创建、编辑、校验、预览并导出幼儿课程包。导出的课程包必须能被课程播放端直接读取和播放。

第一版只做离线制作工具，不接入云端课程库、账号系统、在线同步、在线更新、在线素材市场或多人协作。

### 7.2 编辑端页面与模块

| 模块 | 说明 |
|---|---|
| 启动页 | 加载本地课程工作目录，检查必要目录和 FFmpeg 可用性 |
| 课程管理页 | 新建、打开、复制、删除、搜索课程 |
| 课程编辑页 | 主要工作区，包含基础信息、环节列表、资源面板和预览入口 |
| 基础信息模块 | 编辑课程名称、编号、级别、年龄、主题、封面、预计时长、教学目标 |
| 环节配置模块 | 添加、复制、删除、排序和配置课程环节 |
| 资源管理模块 | 导入、查看、替换、删除课程资源，检查资源缺失 |
| 视频校验/转码模块 | 检查视频格式，调用 FFmpeg 转换为播放端兼容格式 |
| 预览模块 | 按播放端规则预览单环节或完整课程 |
| 导出模块 | 校验课程完整性，生成课程包 |
| 日志与异常提示模块 | 显示保存、校验、转码、导出过程中的结果和错误 |

### 7.3 编辑端主流程

```text
打开编辑端
  ↓
进入课程管理页
  ↓
新建课程或打开已有课程草稿
  ↓
填写课程基础信息
  ↓
导入封面、图片、视频资源
  ↓
添加和配置课程环节
  ↓
视频格式校验，必要时转码
  ↓
预览单个环节或完整课程
  ↓
导出前校验
  ↓
导出离线课程包
```

## 8. 幼儿课程通用环节详细需求

### 8.1 第一版互动边界

第一版支持展示型互动：

- 观察提示。
- 操作提示。
- 选择题展示。
- 判断题展示。
- 找一找题面展示。
- 配对题面展示。
- 拖拽类玩法示意展示。
- 倒计时展示。
- 答案揭示。
- 鼓励反馈展示。
- 星星、贴纸、奖励动画展示。

第一版暂不做：

- 不记录孩子个人答案。
- 不做自动判题。
- 不做得分统计。
- 不做学习记录。
- 不做真实拖拽判定。
- 不做真实连线匹配判定。
- 不做多人触控答题。
- 不做题目随机抽取。
- 不做游戏关卡逻辑。
- 不做复杂动画编辑器。
- 不做语音识别或摄像头识别。

### 8.2 环节通用字段

每个课程环节应至少支持：

| 字段 | 说明 |
|---|---|
| `sectionId` | 系统生成，课程内唯一 |
| `order` | 播放顺序 |
| `title` | 环节名称 |
| `type` | 环节类型 |
| `displayMode` | 展示模式 |
| `titleText` | 主标题 |
| `bodyText` | 副标题或说明文字 |
| `teacherTip` | 老师提示语，主要给老师或制作端预览使用 |
| `studentPrompt` | 面向课堂展示的提示语 |
| `resourcePath` | 主资源路径 |
| `extraResources` | 辅助资源列表 |
| `backgroundPath` | 背景图片，可选 |
| `durationSeconds` | 展示时长或预计时长 |
| `autoNext` | 是否自动进入下一环节 |
| `autoNextMode` | 自动切换方式，例如 videoEnd、delay |
| `countdownSeconds` | 倒计时时长 |
| `interaction` | 展示型互动配置 |
| `enterAnimation` | 进入动画 |
| `exitAnimation` | 退出动画 |
| `editorNote` | 制作端内部备注，不在播放端展示 |

### 8.3 课前准备

用途：帮助老师确认本节课所需材料、设备和课堂状态。

适用内容：

- 教具清单。
- 学生材料。
- 老师演示材料。
- 桌面摆放要求。
- 设备播放检查。
- 课程注意事项。

播放端表现：

- 默认作为课程开始前的老师检查页。
- 可由老师点击开始上课进入开场导入。
- 若设置为仅老师可见，不进入儿童主播放流程。

编辑端要求：

- 支持添加、删除、排序准备项。
- 每个准备项可配置文字和图片。
- 支持设置是否进入正式播放流程。

验收标准：

- 编辑端可创建并保存课前准备环节。
- 播放端可正确展示准备清单。
- 设置仅老师可见时，不出现在儿童主播放流程中。

### 8.4 开场导入

用途：吸引孩子注意力，建立课堂情境。

适用内容：

- 主题引入。
- 情境故事开头。
- 角色出场。
- 问题悬念。
- 主题图片观察。
- 简短热视频。

播放端表现：

- 优先展示视频。
- 未配置视频时展示图片和文字。
- 视频结束后根据 `autoNext` 决定是否自动切换。
- 可显示大号下一步按钮。

编辑端要求：

- 主视频和主图片至少配置一个。
- 支持配置视频是否循环。
- 文本过长时给出提示。
- 导出前校验视频格式。

验收标准：

- 有视频时优先播放视频。
- 无视频但有图片时正常展示图片和文字。
- 自动切换和手动切换均可正常工作。

### 8.5 目标呈现

用途：用孩子能理解的方式呈现本节课任务。

适用内容：

- 本节课学习目标。
- 本节课完成任务。
- 操作成果预览。
- 课堂挑战说明。
- 规则简述。

播放端表现：

- 目标条目建议 1 至 3 条。
- 支持逐条出现。
- 不记录目标完成状态。

编辑端要求：

- 支持添加目标条目。
- 建议最多 3 条。
- 支持为每条目标配置简短文字和图片。

验收标准：

- 目标文字清晰，不被图片遮挡。
- 逐条显示顺序正确。
- 未配置图片时仍可只展示文字。

### 8.6 内容讲解

用途：讲解新知识、新规则、新方法或关键概念。

适用内容：

- 新概念说明。
- 规则讲解。
- 方法讲解。
- 图形、颜色、数量、空间关系说明。
- 魔方步骤说明。
- 思维训练规则说明。

播放端表现：

- 支持视频讲解或图片分页讲解。
- 分页讲解时老师可点击上一页、下一页。
- 重点提示可作为醒目文字条展示。
- 第一版不做画面内热点点击讲解。

编辑端要求：

- 支持选择视频讲解或图片讲解。
- 图片讲解支持多张图片排序。
- 每页可配置短说明文字。
- 导出前校验所有资源。

验收标准：

- 多页图片按配置顺序展示。
- 视频可暂停、继续、重播。
- 老师手动翻页不会跳错页。

### 8.7 示范观察

用途：让孩子观察老师示范、步骤变化、操作细节或最终结果。

适用内容：

- 老师操作示范。
- 分步骤动作。
- 前后变化观察。
- 正确示例与错误示例对比。
- 成果样例展示。
- 关键点观察。

播放端表现：

- 支持播放示范视频。
- 支持分步骤图片展示，并显示步骤编号。
- 可提供再看一次按钮。
- 第一版重点标记采用文字或整图展示，不做图片区域框选编辑。

编辑端要求：

- 支持配置视频示范或分步图片示范。
- 分步图片可调整顺序。
- 支持正确/错误示例并排展示。

验收标准：

- 视频示范可正常重播。
- 分步示范步骤编号正确。
- 自动进入下一环节时切换准确。

### 8.8 课堂练习

用途：让孩子跟随老师完成课堂操作、练习或思考。

适用内容：

- 动手操作练习。
- 跟做练习。
- 小组练习。
- 口头回答。
- 限时完成任务。
- 魔方步骤练习。
- 思维题尝试。

播放端表现：

- 展示练习任务和操作提示。
- 老师点击开始练习后启动倒计时。
- 倒计时结束后显示时间到或自定义提示。
- 第一版不采集孩子练习结果。

编辑端要求：

- 支持配置倒计时时长。
- 支持不开启倒计时，仅展示练习任务。
- 倒计时时长建议 5 秒至 600 秒。

验收标准：

- 倒计时显示准确。
- 未启用倒计时时，练习页正常展示。
- 老师可手动进入下一环节。

### 8.9 游戏巩固

用途：通过轻量游戏化题面帮助孩子复习和巩固课堂内容。

适用内容：

- 选择题。
- 判断题。
- 找一找。
- 看图说答案。
- 配对。
- 拖拽玩法示意。
- 课堂抢答。
- 复习挑战。

播放端表现：

- 第一版按展示型互动实现。
- 选择题展示题干和选项，老师组织孩子回答。
- 判断题展示对/错选项，老师组织孩子判断。
- 找一找展示图片和寻找目标，不做点击区域判定。
- 配对展示左右两组内容，不做真实连线判定。
- 拖拽类只展示玩法示意或目标状态。
- 老师可点击显示答案和显示反馈。

编辑端要求：

- 支持选择游戏类型。
- 不同游戏类型显示对应配置表单。
- 选择题至少 2 个选项，建议最多 4 个。
- 判断题固定为正确/错误两个选项。
- 未配置答案时，导出前提示。

验收标准：

- 各游戏类型可保存、预览、导出。
- 播放端题干、选项、图片展示正确。
- 不出现需要孩子真实点击、拖拽后才能继续的流程阻塞。

### 8.10 结果反馈

用途：给予孩子课堂完成反馈和鼓励。

适用内容：

- 完成祝贺。
- 星星奖励。
- 贴纸奖励。
- 奖杯展示。
- 鼓励语。
- 课堂成果回顾。
- 下一次课程期待。

播放端表现：

- 展示奖励画面和鼓励文字。
- 可播放简单动画，如星星出现、贴纸弹出、淡入淡出。
- 老师可返回课程列表或重播课程。
- 第一版不保存孩子个人奖励记录。

编辑端要求：

- 支持选择奖励类型。
- 支持配置奖励图片和鼓励文字。
- 未配置奖励图片时可使用系统默认奖励样式。

验收标准：

- 播放端可稳定展示奖励画面。
- 默认奖励样式可在无图片时正常显示。
- 返回课程列表和重播课程按钮可正常工作。

### 8.11 课堂收束/延伸

用途：帮助老师完成课堂总结、家庭延伸或下节课预告。

适用内容：

- 本节课回顾。
- 课堂成果展示。
- 回家练习提示。
- 亲子互动建议。
- 下节课预告。
- 材料收纳提醒。

播放端表现：

- 展示简短总结和延伸任务。
- 可展示一张延伸任务图片。
- 老师可手动结束课程或返回课程列表。
- 第一版不做家庭作业提交、不做家长端同步。

编辑端要求：

- 支持配置总结、延伸任务和下节课预告。
- 支持设置该环节是否启用。
- 文本过长时给出提示。

验收标准：

- 启用时播放端按配置顺序展示。
- 禁用时不影响课程正常结束。
- 返回课程列表按钮可正常使用。

## 9. 课程包与 JSON 配置规范

### 9.1 课程包目标

课程包用于统一编辑端导出和播放端读取。课程包必须：

- 可完全离线运行。
- 可复制到任意本地目录。
- 不依赖制作端程序。
- 不依赖外部绝对路径。
- 结构简单、稳定、可人工检查。

### 9.2 单个课程包目录结构

```text
Courses/
  cube_level1_lesson01/
    manifest.json
    course.json
    cover.jpg
    thumbnail.jpg
    resources/
      videos/
        intro.mp4
        demo_step_01.mp4
      images/
        goal.jpg
        step_01.png
        reward.png
      thumbnails/
        intro.jpg
    export_log.txt
```

### 9.3 必需文件

| 文件/目录 | 是否必需 | 说明 |
|---|---|---|
| `manifest.json` | 是 | 课程包清单，用于播放端快速扫描课程列表 |
| `course.json` | 是 | 完整课程配置，用于课程播放 |
| `resources/` | 是 | 课程资源目录 |
| 课程封面图片 | 是 | 路径由 `manifest.json.coverPath` 指定 |

### 9.4 manifest.json 示例

```json
{
  "packageFormatVersion": "1.0.0",
  "courseId": "cube_level1_lesson01",
  "courseCode": "CUBE-L1-001",
  "courseTitle": "魔方启蒙第一课",
  "level": "Level 1",
  "ageRange": "4-6",
  "category": "魔方",
  "theme": "认识魔方",

"resourcePath": "resources/videos/intro.mp4"

"resourcePath": "E:\\Courses\\intro.mp4"

"resourcePath": "../shared/intro.mp4"
```

### 9.12 文件命名规则

建议：

- 文件名使用小写英文字母、数字、下划线或短横线。
- 不建议使用空格。
- 不建议使用中文文件名。
- 同一课程包内资源文件名不得重复到造成混淆。

## 10. 资源与视频规范

### 10.1 支持资源类型

| 类型 | 扩展名 | 用途 |
|---|---|---|
| 视频 | `.mp4` | 开场导入、内容讲解、示范观察、课堂练习 |
| 图片 | `.jpg` / `.jpeg` | 封面、缩略图、课堂展示图 |
| 图片 | `.png` | 透明背景图、奖励图、贴纸图、步骤图 |
| 配置 | `.json` | 课程包清单和课程配置 |

第一版暂不支持：

- 单独音频文件。
- 在线视频地址。
- 网络图片地址。
- XML 配置。
- 外部字幕文件。
- HEVC / H.265 视频。
- WebM、AVI、MOV 等非标准播放资源。

### 10.2 图片规范

- 支持 JPG、JPEG、PNG。
- 色彩空间建议使用 sRGB。
- 图片应提前压缩，不应依赖播放端实时处理大图。
- 图片方向必须在导出前烘焙正确，不依赖 EXIF 方向信息。
- 单张图片长边建议不超过 1920px。
- 老旧电脑兼容优先时，课堂展示图建议使用 1280x720。
- 单张图片文件大小建议不超过 5MB。
- 有透明通道需求时使用 PNG。
- 无透明通道的照片类图片优先使用 JPG。

封面图建议：

- 尺寸：1280x720。
- 比例：16:9。
- 格式：JPG。
- 文件大小：500KB 至 2MB。

缩略图建议：

- 尺寸：320x180 或 400x225。
- 比例：16:9。
- 格式：JPG。
- 文件大小不超过 300KB。

### 10.3 视频规范

课程视频必须满足：

- 容器：MP4。
- 视频编码：H.264 / AVC。
- 音频编码：AAC-LC，或无音频。
- 分辨率：优先 1280x720。
- 帧率：25fps 或 30fps。
- 像素格式：yuv420p。
- 扫描方式：逐行扫描。
- 视频方向：导出时烘焙为正常方向，不依赖旋转元数据。
- 音频采样率：44100Hz 或 48000Hz。
- 声道：双声道或单声道。
- 音频码率：96k 至 160k。

兼容建议：

- H.264 Profile 使用 Main 或 Baseline。
- H.264 Level 建议不高于 3.1。
- 不使用 H.265 / HEVC。
- 不使用 10-bit 视频。
- 不使用 yuv422p、yuv444p。
- 不使用可变帧率视频。
- 不使用 2K、4K 分辨率视频。

### 10.4 视频码率建议

| 视频类型 | 分辨率 | 帧率 | 视频码率建议 |
|---|---|---|---|
| 普通课堂视频 | 1280x720 | 25fps | 1500k 至 2500k |
| 普通课堂视频 | 1280x720 | 30fps | 1800k 至 3000k |
| 画面变化少的视频 | 1280x720 | 25/30fps | 1200k 至 1800k |
| 动作较多的视频 | 1280x720 | 25/30fps | 2500k 至 3500k |

第一版默认建议：

- 视频码率：2500k。
- 最大码率：3000k。
- 缓冲区：6000k。
- 音频码率：128k。

### 10.5 视频时长建议

- 单个视频建议不超过 5 分钟。
- 开场导入视频建议 30 至 90 秒。
- 示范观察视频建议按步骤拆分。
- 长视频应优先拆成多个课程环节，方便老师控制和重播。

### 10.6 FFmpeg 转码参数

编辑端第一版从 `StreamingAssets/ffmpeg` 目录加载或调用 FFmpeg 相关依赖。打包时应确保该目录随编辑端程序一起发布。

推荐 720p / 25fps：

```bash
ffmpeg -i input.mp4 ^
  -vf "scale=1280:720:force_original_aspect_ratio=decrease,pad=1280:720:(ow-iw)/2:(oh-ih)/2,fps=25,format=yuv420p" ^
  -c:v libx264 ^
  -profile:v main ^
  -level 3.1 ^
  -preset medium ^
  -b:v 2500k ^
  -maxrate 3000k ^
  -bufsize 6000k ^
  -pix_fmt yuv420p ^
  -movflags +faststart ^
  -c:a aac ^
  -b:a 128k ^
  -ar 44100 ^
  -ac 2 ^
  output_720p25.mp4
```

推荐 720p / 30fps：

```bash
ffmpeg -i input.mp4 ^
  -vf "scale=1280:720:force_original_aspect_ratio=decrease,pad=1280:720:(ow-iw)/2:(oh-ih)/2,fps=30,format=yuv420p" ^
  -c:v libx264 ^
  -profile:v main ^
  -level 3.1 ^
  -preset medium ^
  -b:v 2500k ^
  -maxrate 3000k ^
  -bufsize 6000k ^
  -pix_fmt yuv420p ^
  -movflags +faststart ^
  -c:a aac ^
  -b:a 128k ^
  -ar 44100 ^
  -ac 2 ^
  output_720p30.mp4
```

老旧电脑优先兼容参数：

```bash
ffmpeg -i input.mp4 ^
  -vf "scale=1280:720:force_original_aspect_ratio=decrease,pad=1280:720:(ow-iw)/2:(oh-ih)/2,fps=25,format=yuv420p" ^
  -c:v libx264 ^
  -profile:v baseline ^
  -level 3.1 ^
  -preset medium ^
  -b:v 2000k ^
  -maxrate 2500k ^
  -bufsize 5000k ^
  -pix_fmt yuv420p ^
  -movflags +faststart ^
  -c:a aac ^
  -b:a 128k ^
  -ar 44100 ^
  -ac 2 ^
  output_compatible.mp4
```

## 11. 版本兼容与读取规则

### 11.1 版本字段

课程包必须包含：

- `manifest.json.packageFormatVersion`。
- `manifest.json.minPlayerVersion`。
- `course.json.schemaVersion`。

第一版开发阶段允许协议字段边开发边调整。每次调整协议时，必须同步更新：

- 播放端解析逻辑。
- 编辑端导出逻辑。
- 示例 `manifest.json`。
- 示例 `course.json`。
- 本文档字段说明。

### 11.2 播放端兼容策略

- 如果 `packageFormatVersion` 主版本号高于播放端支持版本，禁止加载并提示课程包版本过高。
- 如果主版本号一致、小版本号高于播放端支持版本，可以尝试加载，但需要提示可能存在兼容风险。
- 如果缺少必填版本字段，禁止加载。
- 播放端必须忽略未知的非必填字段。
- 播放端不得因为 JSON 中出现未知字段而崩溃。

### 11.3 编辑端兼容策略

- 导入旧课程包时，应保留可识别字段。
- 对缺失的新增字段使用默认值。
- 重新导出时应写入当前支持的版本字段。
- 不应静默删除资源文件，除非用户明确执行清理操作。

### 11.4 播放端读取规则

播放端启动或进入课程列表时：

1. 扫描指定课程目录下的一级子文件夹。
2. 判断子文件夹中是否存在 `manifest.json`。
3. 读取并解析 `manifest.json`。
4. 校验必填字段、版本字段和封面路径。
5. 将合法课程加入课程列表。
6. 对不合法课程记录错误并跳过。

播放课程时：

1. 根据 `manifest.json.courseJsonPath` 读取 `course.json`。
2. 校验 `courseId`、`courseCode` 与 `manifest.json` 是否一致。
3. 按 `sections.order` 从小到大排序播放。
4. 根据 `displayMode` 选择视频、图片、文字或互动展示方式。
5. 加载资源前必须检查文件是否存在。
6. 视频播放失败时应显示明确提示，并允许重试、跳过或返回。
7. 图片加载失败时应显示占位提示，不应导致程序崩溃。
8. `autoNext = true` 时，当前环节结束后自动进入下一环节。
9. `autoNext = false` 时，等待老师手动点击下一环节。
10. 老师手动控制优先级高于自动播放逻辑。

### 11.5 编辑端导出规则

编辑端导出课程包时必须：

1. 创建课程包根目录。
2. 写入 `manifest.json`。
3. 写入 `course.json`。
4. 复制课程封面、缩略图和所有引用资源。
5. 将所有资源路径转换为课程包内相对路径。
6. 检查 JSON 中引用的每个资源是否存在。
7. 检查视频格式是否符合规范。
8. 对不符合规范的视频给出转码提示，或调用 FFmpeg 转码。
9. 导出完成后重新从导出目录读取一次课程包，确认播放端可读取。
10. 生成导出结果提示，列出成功项、警告项和错误项。

编辑端不得导出：

- 缺少 `manifest.json` 的课程包。
- 缺少 `course.json` 的课程包。
- 没有任何课程环节的课程包。
- 引用了不存在资源的课程包。
- 包含不兼容视频且未转码成功的课程包。

## 12. 校验规则

### 12.1 JSON 校验

必须校验：

- JSON 文件可被 LitJSON 正常解析。
- JSON 不包含注释。
- JSON 不包含尾随逗号。
- 必填字段存在。
- 字段类型正确。
- `sections` 数组不为空。
- `sectionId` 不重复。
- `order` 不重复。
- `resourcePath` 不为空时文件必须存在。
- `manifest.json.courseId` 与 `course.json.courseId` 一致。
- `manifest.json.courseCode` 与 `course.json.courseCode` 一致。

### 12.2 资源校验

必须校验：

- 所有引用资源都在课程包目录内。
- 所有引用资源文件存在。
- 文件扩展名与资源类型匹配。
- 路径不包含绝对路径。
- 路径不包含 `..`。
- 路径分隔符使用 `/`。
- 文件大小不明显异常。
- 图片可被 Unity 正常加载。
- 视频可被 AVPro Video 正常打开。

### 12.3 视频校验

必须校验：

- 容器为 MP4。
- 视频编码为 H.264。
- 音频编码为 AAC，或无音频。
- 分辨率不高于 1280x720，特殊情况需编辑端警告。
- 帧率为 25fps 或 30fps，或接近该范围并可转码。
- 像素格式为 yuv420p。
- 不是 H.265 / HEVC。
- 不是 10-bit 视频。
- 码率在建议范围内。
- 视频可正常解码首帧。

### 12.4 错误级别

| 级别 | 说明 | 是否允许导出 |
|---|---|---|
| Error | 会导致播放端无法读取或播放 | 否 |
| Warning | 可能影响兼容性或体验 | 可由用户确认后导出 |
| Info | 普通提示 | 是 |

第一版中，视频不兼容建议按 Error 处理，阻止导出。

## 13. 异常处理需求

### 13.1 播放端异常

| 编号 | 场景 | 处理要求 |
|---|---|---|
| CP-ERR-001 | `Courses` 目录不存在 | 显示未找到课程目录提示，允许刷新或退出 |
| CP-ERR-002 | `Courses` 目录为空 | 显示暂无课程提示，允许刷新 |
| CP-ERR-003 | 课程包缺少 `manifest.json` | 跳过该课程包 |
| CP-ERR-004 | `manifest.json` 或 `course.json` 格式错误 | 跳过该课程包，并记录日志 |
| CP-ERR-005 | 课程必要字段缺失 | 标记为无效课程，不进入播放 |
| CP-ERR-006 | 课程封面缺失 | 使用默认封面 |
| CP-ERR-007 | 环节资源文件缺失 | 播放前提示资源缺失，允许跳过或返回列表 |
| CP-ERR-008 | 视频文件无法打开 | 显示播放失败提示，允许重试、跳过或返回 |
| CP-ERR-009 | 视频播放中断 | 停止当前播放，显示失败提示，允许重试 |
| CP-ERR-010 | 图片加载失败 | 显示图片加载失败提示，允许跳过 |
| CP-ERR-011 | 播放到最后一个环节 | 显示课程结束状态，允许重播或返回列表 |
| CP-ERR-012 | 快速连续点击切换环节 | 避免重复加载导致卡死或状态错乱 |
| CP-ERR-013 | 本地记录写入失败 | 不影响课程播放，仅跳过记录保存 |
| CP-ERR-014 | 低性能设备加载较慢 | 显示加载状态，不应长时间黑屏无提示 |

### 13.2 编辑端异常

| 编号 | 场景 | 处理要求 |
|---|---|---|
| CE-ERR-001 | 课程 JSON 损坏 | 提示课程无法打开，并保留原文件，不自动覆盖 |
| CE-ERR-002 | 必填信息缺失 | 标记具体字段，允许保存草稿，阻止导出 |
| CE-ERR-003 | 环节为空 | 允许保存草稿，阻止导出 |
| CE-ERR-004 | 环节资源缺失 | 标记对应环节，阻止导出 |
| CE-ERR-005 | 视频格式不兼容 | 标记视频状态，提示转码，阻止导出 |
| CE-ERR-006 | FFmpeg 不存在 | 提示无法自动转码，允许继续编辑和保存草稿 |
| CE-ERR-007 | 转码失败 | 显示失败原因，保留原视频，不替换引用 |
| CE-ERR-008 | 磁盘空间不足 | 停止导入、转码或导出，提示清理空间 |
| CE-ERR-009 | 文件被占用 | 提示关闭占用文件的软件后重试 |
| CE-ERR-010 | 路径权限不足 | 提示更换导出目录或检查权限 |
| CE-ERR-011 | 图片格式不支持 | 拒绝导入并提示仅支持 JPG、JPEG、PNG |
| CE-ERR-012 | 预览失败 | 显示当前环节和资源路径，便于定位 |
| CE-ERR-013 | 软件异常退出 | 下次启动时尝试读取最近自动保存草稿 |

### 13.3 日志要求

- 播放端应记录课程包解析失败、资源缺失、视频播放失败等错误。
- 编辑端应记录保存、导入、转码、校验、导出过程中的错误和警告。
- 日志应存放在本地目录。
- 日志写入失败不得影响核心流程。
- 日志中不记录学生个人信息。

## 14. 非功能需求

### 14.1 离线要求

- 播放端和编辑端在无网络环境下必须可启动。
- 课程创建、保存、预览、导出、播放均不得依赖网络。
- 所有课程资源必须来自本地文件。

### 14.2 稳定性要求

- 损坏课程包不应导致播放端崩溃。
- 单个资源加载失败不应导致整个程序退出。
- 转码失败不应破坏原视频。
- 导出失败不应破坏课程草稿。
- 保存过程中异常中断不应破坏最近一次成功保存的草稿。

### 14.3 性能与兼容要求

- 优先支持 1280x720 课堂播放。
- 支持 1920x1080 分辨率显示。
- 老旧 Windows 笔记本播放推荐格式 MP4 时，应避免频繁黑屏、卡死或闪退。
- 图片不应过大，避免加载时明显卡顿。
- 单个视频建议不超过 5 分钟。

### 14.4 可用性要求

- 播放端核心按钮必须适合触屏点击。
- 播放端不应要求老师理解文件结构。
- 高风险操作必须二次确认。
- 错误提示应面向老师可理解，避免只显示技术异常。
- 编辑端导出失败时，应明确指出失败项和修复建议。

## 16. 验收标准

### 16.1 课程包验收

一个课程包视为合格，必须满足：

- 课程包可复制到任意本地目录后离线运行。
- 课程包根目录包含 `manifest.json` 和 `course.json`。
- 播放端能在课程列表中显示课程名称、封面、级别和主题。
- 播放端能打开课程并按环节顺序播放。
- 所有视频、图片、文字内容均能正常展示。
- 删除网络连接后不影响课程播放。
- 课程包路径改变后仍可正常读取。

### 16.2 播放端验收

- 在无网络环境下，播放端可正常启动。
- 程序同级 `Courses` 目录中放入有效课程包后，播放端可扫描并展示课程。
- 每个有效课程至少正确展示封面、课程名称、课程级别、课程主题。
- 删除或新增课程包后，点击刷新可更新课程列表。
- 点击课程后可进入课程播放页。
- 课程按 `course.json` 中配置的环节顺序播放。
- MP4 视频可正常加载、播放、暂停、继续、重播。
- PNG、JPG、JPEG 图片可正常展示。
- 当前环节名称、环节序号和总环节数显示正确。
- 自动切换和手动切换配置生效。
- 播放到最后一个环节后，可重播课程或返回课程列表。
- 环节列表可打开，并能跳转到指定环节。
- 控制面板可隐藏和再次展开。
- 互动展示不要求保存学生答案或统计成绩。
- 返回课程列表、退出程序等高风险操作有二次确认。
- 遇到损坏课程包时跳过并提示，不影响其他课程。
- 视频播放失败时不崩溃，并给出可理解提示。

### 16.3 编辑端验收

- 在无网络环境下，编辑端可正常启动。
- 用户可新建一门课程并填写完整基础信息。
- 用户可添加至少 7 类通用环节中的任意一种。
- 用户可调整环节顺序。
- 用户可为环节绑定图片、视频和文字内容。
- 用户可保存草稿，再次打开后内容保持一致。
- 用户可导入 JPG、JPEG、PNG、MP4。
- 导入后的资源被复制到课程草稿目录。
- 课程配置中记录的是相对路径。
- 删除或移动资源后，编辑端能检测到资源缺失。
- 编辑端能识别 MP4/H.264/AAC 视频。
- 编辑端能提示不兼容视频。
- 本地 FFmpeg 可用时，编辑端能将不兼容视频转为推荐格式。
- 转码后视频能在预览中播放。
- 用户可预览单个环节。
- 用户可按顺序预览完整课程。
- 导出课程包包含 `manifest.json`、`course.json` 和所有被引用资源。
- 导出课程包中不存在对制作电脑绝对路径的依赖。
- 导出课程包复制到播放端 `Courses` 目录后，播放端能展示并播放。

### 16.4 视频验收

视频资源必须满足：

- MP4 / H.264 / AAC。
- 720p 优先。
- 25fps 或 30fps。
- 像素格式 yuv420p。
- 老旧 Windows 笔记本上播放无黑屏、无明显卡顿、无音画不同步。
- 单个视频可暂停、继续、重播。
- 视频结束后能按 `autoNext` 正确切换环节。

### 16.5 最低测试课程包

第一版至少准备 3 个测试课程包：

- 纯图片和文字课程包。
- 多段视频课程包。
- 图片、视频、互动提示混合课程包。

至少在以下环境验证：

- Windows 触屏一体机。
- Windows 普通电脑。
- 一台性能较弱或较老的 Windows 笔记本。

所有测试课程包必须能完成从编辑端导出到播放端播放的完整闭环。

## 17. 开发拆分建议

### 17.1 播放端开发包

优先顺序：

1. 课程包扫描与 manifest 解析。
2. 课程列表 UI。
3. course.json 解析与环节模型。
4. 视频、图片、文字播放。
5. 老师控制面板。
6. 环节列表跳转。
7. 展示型互动。
8. 异常提示与日志。
9. 触屏与全屏适配。

### 17.2 编辑端开发包

优先顺序：

1. 课程草稿管理。
2. 基础信息编辑。
3. 环节增删改排。
4. 资源导入与复制。
5. 课程预览。
6. 视频校验。
7. FFmpeg 转码。
8. 导出课程包。
9. 导出后自校验。
10. 异常提示与日志。

### 17.3 公共模块

播放端和编辑端应共享：

- 课程包路径工具。
- JSON 数据模型。
- LitJSON 解析与写入封装。
- 资源相对路径校验。
- 课程环节枚举。
- 展示模式枚举。
- 视频格式校验结果模型。
- 错误码和日志格式。

## 18. 已确认与待确认问题

### 18.1 已确认问题

| 问题 | 结论 |
|---|---|
| 最低 Windows 版本 | 支持 Windows 7 及以上。 |
| FFmpeg 集成方式 | FFmpeg DLL 和相关依赖放入 `StreamingAssets/ffmpeg` 目录。 |
| 播放端默认显示模式 | 默认全屏。 |
| 第一版互动边界 | 只做展示型互动。 |
| 课程包协议是否冻结 | 第一版开发期间不冻结，可边做边改，但必须同步更新代码和文档。 |
| 是否已有视觉稿 | 暂无正式视觉稿，先输出低保真视觉稿用于第一版开发。 |

### 18.2 待确认问题

- 课程分类是否固定枚举，还是由编辑端自由输入。
- 课程封面是否必须由编辑端自动压缩到推荐尺寸。
- 第一版是否允许 Warning 级视频问题由用户确认后导出，本文档建议不允许。
- 后续是否需要基于低保真视觉稿制作正式 UI 视觉规范，包括颜色、字体、图标和组件尺寸。
