# 幼儿课程课件系统设计大纲

## 1. 项目定位

本项目是一套面向幼儿线下课堂的离线课程课件系统，可用于魔方、思维训练、启蒙认知、动手操作等多种幼儿课程。

系统采用一个 Unity 工程开发，在同一个工程中打包两个独立程序：

- 课程播放端：用于课堂现场播放课程。
- 课程制作端：用于配置课程内容并导出离线课程包。

项目只考虑离线版本，不设计云端课程库、在线下载、在线更新、账号系统或联网同步。

## 2. 使用对象

- 幼儿课堂老师
- 课程制作老师

## 3. 运行环境

- Windows 触屏一体机
- Windows 普通电脑
- 老旧 Windows 笔记本
- 本地离线课程资源包

## 7. 幼儿课程通用环节

幼儿课程环节不宜过多，建议保留少量高通用性环节。每个环节只定义课堂用途和内容形式，不绑定具体课程品类。

### 7.1 开场导入

- 用途：吸引孩子注意力，进入课堂状态。
- 内容形式：短视频、图片、标题文字。

### 7.2 目标呈现

- 用途：告诉孩子本节课要完成什么。
- 内容形式：图片、标题文字、简短说明。

### 7.3 内容讲解

- 用途：讲解本节课的新知识、新规则或新方法。
- 内容形式：视频、图片、文字提示。

### 7.4 示范观察

- 用途：让孩子观察老师示范、步骤变化或操作结果。
- 内容形式：视频、分步图片、重点提示。

### 7.5 课堂练习

- 用途：让孩子跟随老师完成课堂操作。
- 内容形式：练习提示、倒计时、图片、视频。

### 7.6 游戏巩固

- 用途：通过小游戏复习和巩固课堂内容。
- 内容形式：选择题、判断题、找一找、配对、拖拽类互动。

### 7.7 结果反馈

- 用途：给予孩子课堂完成反馈和鼓励。
- 内容形式：结果画面、奖励动画、星星、贴纸、鼓励文字。

## 9. 视频兼容方案

为了避免老旧电脑，尤其是部分老旧 AMD CPU 笔记本出现视频卡顿、黑屏或无法播放的问题，课程制作流程中需要加入视频转码要求。

### 9.1 视频格式要求

- MP4 容器
- H.264 视频编码
- AAC 音频轨道可保留在视频内部
- 分辨率优先使用 1280x720
- 帧率优先使用 25fps 或 30fps
- 码率控制在适合老旧电脑播放的范围

### 9.2 视频转换工具

- FFmpeg

FFmpeg 作为课程制作流程中的视频转换工具，用于把来源不统一的视频统一转换为播放端兼容格式。

### 9.3 制作端相关功能

- 检查视频格式
- 提示不兼容视频
- 调用或引导使用 FFmpeg 转换视频
- 导出课程包前确认视频已符合格式要求

## 13. 第一版设计原则

- 只做离线版本
- 一个 Unity 工程打包两个程序
- 优先保证视频播放稳定
- 优先兼容老旧 Windows 电脑
- 优先支持老师线下带课
- 课程制作尽量可视化
- 课程包结构尽量简单
- 播放端适合触屏操作
- 功能保持克制，先做稳定可用版本
